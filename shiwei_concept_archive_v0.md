# 师维理念沉淀 V0

BOUNDARY=NO_RUNTIME_APPLY
SOURCE=当前会话关于备课室、备课本、资料室、档案室与驻场智能体权属的产品判断

## 1. 总定位

```text
师维 = 学校里的 AI 工作社区
```

当前美术教师线只是首个角色样板，不应把系统写死成单一美术教师工具。

```text
身份决定空间
空间决定 Agent
Agent 承办对应事务
```

## 2. 权属关系

```text
师维 = 总平台 / 总品牌
备课室 = 一个工作空间
小备 / 小教 / 小评 / 小美 / 小管 = 驻场智能体
```

备课室顶部可以显示当前空间与当前看板：

```text
备课室 · 周课表
备课室 · 班级排课
备课室 · 学期规划
备课室 · 备课本
```

但备课室不应表达成“小教管辖”。备课室主助理建议为小备，小教只在课堂执行、课前调度、周课表、班级进度等专项场景中协同出现；小管可在班级、课表、校历、排课调整中协同。

## 3. 驻场智能体边界

| 场景 | 建议智能体 |
| --- | --- |
| 课时设计、学习单、课前包、备课本 | 小备 |
| 上课流程、课中提醒、班级进度、周课表推送 | 小教 |
| 评价量规、作品评价、评语 | 小评 |
| 素材美化、PPT、图片生成 | 小美 |
| 班级、课表、校历、排课调整 | 小管 / 小教协同 |

当前原则：

```text
空间归备课室
主助理是小备
排课、课前执行由小教/小管协同
```

## 4. 三类空间边界

最清楚的边界：

```text
资料室 = 备课前可调用的东西
备课本 = 本学期正在发生的工作
档案室 = 已经发生并确认沉淀的东西
```

一句话：

```text
资料室：备用
备课本：正在用
档案室：用完留下
```

对应关系：

```text
资料室管可用资源
备课本管当前过程
档案室管历史证据
```

## 5. 备课本定义

```text
备课本 = 本学期的过程性工作本
```

它不是普通文档，也不是资料库。它更像：

```text
结构化思维导图 + 抽屉式工作台 + 学期过程沉淀
```

从学期规划开始，一层层展开到：

```text
学期
→ 单元
→ 课时
→ 班级课时
→ 课前包
→ 课堂记录
→ 学生作品 / 评价证据
→ 课后反思
```

核心价值：

```text
让老师知道本学期每一节课做到哪里了。
```

## 6. 备课本建议结构

```text
2026春学期 · 三年级美术备课本

├─ 学期规划
│  ├─ 学期目标
│  ├─ 单元安排
│  └─ 班级进度与排课
│
├─ 单元一：色彩单元
│  ├─ 1-1 认识色彩
│  ├─ 1-2 色彩的感觉
│  └─ 1-3 色彩的对比
│
└─ 单元二：造型表现单元
   ├─ 2-1 形体与空间
   └─ 2-2 材料的奇妙
```

每个节点点开后，是抽屉式工作内容：

```text
课时设计
学习单
评价量规
资源参考
课前包
班级进度
课堂记录
课后反思
```

## 7. 备课本页面 V0 方向

```text
左侧：学期 / 单元 / 课时目录
中间：当前节点工作区
右侧：资料、建议、记录、课前包抽屉
底部：对小备说一句
```

左侧目录像思维导图，但要规整；中间展示当前课时或当前单元的工作状态；右侧按需打开资料抽屉、课前包抽屉、学生记录抽屉、小备建议抽屉、历史版本抽屉。

## 8. 资料室定义

资料室是“拿来用”的地方，教师端不建议直接叫“知识库”。

可放：

```text
教材
课标
优秀课例
图片素材
范作
评价语
常用模板
学校资料
过往可复用案例
```

建议内部结构：

```text
资料室
├─ 教材课标
├─ 优秀课例
├─ 图片素材
├─ 范作资源
├─ 评价语库
└─ 校本资料
```

## 9. 档案室定义

档案室是“已经发生过并沉淀下来”的地方。

关键词：

```text
已完成
已确认
可追溯
有证据
可归档
```

可放：

```text
已完成备课
课堂实录
学生作品证据
评价记录
课后反思
公开课材料
教研成果
导出的成果包
```

建议内部结构：

```text
档案室
├─ 已完成备课
├─ 课堂记录
├─ 学生作品证据
├─ 教学反思
├─ 公开课材料
└─ 成果包
```

## 10. 容易混淆的边界

### 教案

| 内容 | 放哪里 |
| --- | --- |
| 网上/市级/省级优秀教案 | 资料室 |
| 本学期自己正在完成的教案 | 备课本 |
| 学期结束后归档的最终教案 | 档案室 |
| 从归档教案中提炼出的可复用模板 | 资料室 |

### 课堂记录

| 内容 | 放哪里 |
| --- | --- |
| 本周刚记下的课堂观察 | 备课本 |
| 学期末整理后的课堂证据 | 档案室 |
| 被提炼成教学经验的记录 | 资料室 / 经验库 |

### 资源

| 内容 | 放哪里 |
| --- | --- |
| 可复用图片素材 | 资料室 |
| 本节课选用的图片素材 | 备课本 |
| 本节课最终使用并留痕的素材包 | 档案室 |

## 11. 流转闭环

```text
资料室
↓
被选用
↓
备课本
↓
上课 / 修改 / 评价 / 记录
↓
档案室
↓
提炼成可复用经验
↓
资料室
```

也可以理解为：

```text
资料室：从外面拿进来
备课本：在本学期用起来
档案室：用完以后留下来
资料室：从留下来的东西里再提炼出可复用内容
```

## 12. 备课室内部命名建议

```text
备课室
├─ 备课本
├─ 周课表
├─ 班级进度与排课
├─ 学期规划
└─ 课前包
```

下一阶段建议先设计：

```text
备课本 V0
```

先不急着展开资料室和档案室。备课本是主工作流，资料室和档案室是它两侧的支撑。

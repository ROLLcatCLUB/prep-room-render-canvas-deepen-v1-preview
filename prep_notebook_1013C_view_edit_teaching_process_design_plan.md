## 1. 核心判断

备课本不是一个“AI 生成教案页面”。它应该是教师本学期正在使用的过程性工作本。

下一步必须把备课本分成两种状态：

```text
查看状态 = 老师顺畅读一份可上课的教学设计
编辑状态 = 老师和小备集中修改某一段、某一环节、某一份材料
```

现在页面里很多“生成候选、继续打磨、确认、查看影响”其实属于编辑状态。它们如果常驻在查看状态，会破坏老师阅读教案的连续性，也会让边界变混：老师分不清“我现在是在看一份课，还是在改一份课”。

新的备课本应该像这样：

```text
默认打开：我在看这节课
点编辑：我在改这一段
点教学过程某环节：我在推敲这个环节为什么这样设计
对小备说一句：小备把意图落到具体段落和影响范围
确认后：仍是候选确认，不进入正式写回
```

## 2. 市面应用检索结论

本轮检索的产品和资料包括：

- MagicSchool Lesson Plan Generator: 强在标准对齐、主题/目标定制、快速生成结构化教案。参考: https://www.magicschool.ai/tools/lesson-plan
- Eduaide Lesson Planning Workspace: 强在 first draft 之后继续 revise、differentiate、evaluate、add visuals、export。参考: https://www.eduaide.ai/solutions/lesson-planning
- Curipod: 强在 teacher-led lessons，学生读、写、讨论，课堂过程会产生即时反馈和报告。参考: https://curipod.com/lang/en-US
- Curipod Safe AI: 强调教师发起、教师可编辑、学生不独自面对 AI、AI 填充在人设计的课堂结构里。参考: https://curipod.com/safe-ai-in-curipod
- 希沃教学大模型: 强在学科知识库、AI 生成教学目标/教学思路/互动课件、教师可调整编辑，强调不是直接给完整答案。参考: https://www.seewo.com/product/detail/AI
- 星火教师助手: 强在对话式自然交互，生成大单元教学规划、教学设计、教学课件。参考: https://edu.iflytek.com/solution/school/teachers-assistant
- 云知师 AI: 强在入驻国家智慧教育公共服务平台，定位教师教学智能工具，强调模板化操作、智能引导、课标与学情适配。参考: https://www.ecnu.edu.cn/info/1094/71441.htm
- SchoolAI 案例: 强在教师实时参与、交互可观测、护栏、智能体图谱和“不是直接给答案”。参考: https://openai.com/zh-Hans-CN/index/schoolai/

市场上较成熟的方向有五类：

```text
工具箱型：教案、量规、学习单、测验、评语等工具很多。
工作流型：先出初稿，再修订、分层、评价、导出。
嵌入型：AI 贴在 Google/Microsoft/课堂文档旁边工作。
互动课堂型：重学生回答、讨论、反馈和课堂证据。
平台全链路型：备课、授课、评价、教研、硬件或学校平台联动。
```

我们应该吸收的不是“工具菜单”，而是这些经验：

```text
1. 工具要颗粒清楚，但藏在字段和段落后面。
2. 初稿不是终点，打磨、分层、评价和材料联动才是关键。
3. AI 要贴着当前工作对象，不要悬空长聊。
4. 教学过程必须落到学生做什么、老师做什么、大屏呈现什么、评价证据在哪里。
5. 任何候选都要有人类教师确认，不直接变成正式结果。
```

我们的差异点应该更明确：

```text
市面常见：帮老师生成内容。
师维备课本：陪老师把一节课设计成立。
```

## 3. 备课本的产品定位

备课本要承接这些真实对象：

```text
飞书正式课表
学期教学工作计划
教材单元与课时目录
资料室材料
学生已有认知与往期课堂记录
本课教学设计
课堂大屏与学习单
评价证据
课后记录与档案室沉淀
```

它不是资料室：

```text
资料室 = 备用资源
备课本 = 正在用的本学期工作本
档案室 = 用完以后留下的证据
```

所以备课本的核心不是“列资源”，而是组织一节课：

```text
为什么教
学生在哪里
本节课要把学生带到哪里
用什么环节推进
老师怎么引导
学生怎么活动
大屏和材料怎么配合
怎么判断达成
课后留下什么
```

## 4. 查看状态设计

查看状态是默认状态。老师打开备课本时，应该先看到一份能读下来的教学设计，而不是一堆编辑工具。

查看状态的主内容建议为：

```text
课题
可上课状态 / 待确认项

一、本课依据
二、学情分析
三、教学目标
四、教学重难点
五、教学准备
六、教学过程
七、学习单与评价
八、课后记录与沉淀
```

查看状态允许出现的轻量信息：

```text
教材
课标
资料室
档案室
待确认
可上课
查看设计说明
进入编辑
```

查看状态不应该常驻出现：

```text
替换本段
并入本段
生成候选
继续打磨
查看影响
field_patch
schema
ViewModel
provider
formal_apply
```

查看状态的右侧栏应该是阅读辅助，不是编辑面板：

```text
资料库
档案室
本课材料
待确认
待沉淀
```

右侧语言要像老师能用的话：

```text
教材原文片段
色彩情感卡片
优秀课例：冷暖色
学习单评价标准待确认
学生优秀作品待收集
```

## 5. 编辑状态设计

编辑状态只在老师主动点“编辑”、点某一段、点某一教学环节，或从底部对小备说一句后出现。

编辑状态要让老师明确知道：

```text
我正在改哪一段
小备为什么建议这样改
这会影响哪些材料、问题串、学习单、大屏、评价证据
哪些内容只是候选
确认前不会正式应用
```

编辑状态主内容可以出现：

```text
当前编辑章节
当前段落原文
小备建议
候选版本
依据
影响范围
教师确认
```

编辑状态按钮可以是：

```text
并入本段
替换本段
再简化
再具体一点
补学习单
补大屏呈现
先放着
```

编辑状态右侧栏从阅读辅助切换为编辑辅助：

```text
当前编辑：教学过程 · 探究环节
小备判断
字段缺口
相关资料
影响范围
候选确认
```

重点是，编辑状态的工具不能散在整页各处。它们应该围绕当前段落出现，这样老师不会被打断。

## 6. 教学过程意图层

教学过程是备课本最关键的地方。它不能只是：

```text
导入 3 分钟
感知 7 分钟
探究 10 分钟
表现 15 分钟
交流展示 5 分钟
```

这只是流程壳。真正的教学设计要解释：

```text
为什么安排这个环节
它承接了上一环节什么
它要让学生发生什么变化
学生此时可能是什么状态
老师此时应该做什么
大屏应该呈现什么
学生手上有没有学习单、教材或材料
这一环节产生什么评价证据
如果学生不理解，老师怎么调整
```

因此每个教学环节背后要有一套结构：

```text
环节名称
时间
环节作用
设计意图
承上启下
学生当前状态
教师动作
学生活动
大屏状态
学习单 / 教材 / 材料状态
评价证据
风险与调整
```

在查看状态中，不要把这些字段全部铺开。查看态只显示顺畅正文：

```text
3. 探究（10分钟）
学生分组观察色卡和生活物品，尝试把颜色分成“温暖、清凉、安静、热烈”等感受类型，并说出理由。
```

旁边保留一个轻按钮：

```text
查看设计说明
```

展开后才看到：

```text
环节作用：帮助学生从生活感受进入色彩情绪表达。
设计意图：避免直接讲冷暖色概念，先通过生活物品和作品建立直观感受。
承上启下：承接感知环节的图片比较，推进到学生自己分类和表达理由。
学生当前状态：学生可能能说出喜欢的颜色，但不一定能说明颜色带来的感受。
教师动作：出示图片，追问“这组颜色给你什么感觉？为什么？”
大屏状态：展示冷暖色图片对比、情绪词卡和活动步骤。
学习单状态：记录“我看到的颜色”和“我感受到的情绪”。
评价证据：学生能用一句话说明颜色与感受之间的关系。
风险与调整：如果学生只说“好看”，教师提供情绪词卡辅助表达。
```

## 7. 1-2《色彩的感觉》的示范结构

本课不应只做“冷暖色知识点讲解”。它应该从学生已有经验出发：

```text
学生可能已经知道常见颜色，也能说出喜欢或不喜欢。
但他们未必能把颜色、情绪、生活场景、作品表达联系起来。
```

本课目标不宜太低：

```text
不能只停留在“认识冷暖色”。
```

也不能超纲：

```text
不能变成专业色彩心理学或复杂配色理论。
```

合理目标是：

```text
让学生感受不同色彩带来的情绪差异；
能用语言描述自己的色彩体验；
能在小练习中有意识地选择色彩表达心情或场景；
能听懂同伴的表达并做简单评价。
```

建议教学过程：

```text
导入 3 分钟：用两组生活图片唤起感受，不直接讲概念。
感知 7 分钟：看作品和生活图，建立色彩与情绪的联系。
探究 10 分钟：用色卡/物品分组，让学生说理由，诊断是否理解。
表现 15 分钟：围绕“我的心中的____”做色彩小练习。
交流展示 5 分钟：学生说色彩选择，同伴听懂并补充。
```

每一环节都要有“设计说明”，并且能联动：

```text
大屏：图片、问题、步骤、计时、展示作品
学习单：观察记录、色彩感受、创作说明、评价小格
教材：需要看原文/范图时才打开，不让教材变成形式
资料室：图片、课例、情绪词卡、色彩卡
评价：学生能否说出关系、能否用色彩表达、能否解释选择
```

## 8. 小备的工作方式

小备不能直接生成整篇教案后让老师接受。它要先做教学判断，再生成候选。

内部推理问题：

```text
1. 这一环节在整节课中的功能是什么？
2. 学生进入这一环节前已经知道什么？
3. 学生可能卡在哪里？
4. 这个活动是为了背景了解、概念建立、技能示范、创作实践，还是评价反馈？
5. 老师此时是讲解、追问、示范、巡视，还是放手？
6. 大屏此时应该呈现什么？
7. 是否应该发学习单，还是先看教材/图片/作品？
8. 这一环节产生什么评价证据？
9. 它如何过渡到下一环节？
10. 这个设计会不会太简单或超出学生认知？
```

对老师显示时，必须说人话：

```text
我建议把“探究”改得更直观一点。
学生现在可能还停在“这个颜色好看”的描述，需要先用色卡分组，帮助他们说出“为什么有这种感觉”。
这会影响两处：学习单要加一个记录格，大屏要准备冷暖色对比图。
我先放成候选，等你确认。
```

不要显示：

```text
正在执行 field_patch
target_layer=l2_lesson_execution
formal_apply_performed=false
schema validation pass
```

这些只留在工程内部。
